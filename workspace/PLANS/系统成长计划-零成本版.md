## P2 — 覆盖广度（免费数据源）

### 2.1 行业轮动分析

**状态：** ✅ 已完成（2026-05-14）
- **接口：** `ak.stock_board_industry_summary_ths()` — 同花顺行业实时指数
- **命令：** `python akshare_data.py sector`
- **覆盖：** 90个行业，含涨跌幅、成交额、资金净流入、领涨股
- **成本：** 0 元
- 不再依赖手工维护的 sector_map.json

**已弃用的原方案：** 本地行业映射表（20行业~190标的）+ Tencent GT API — 已被同花顺90行业全覆盖替代。

**实现路径：**
```
Orobas 收到请求
    → 读取本地 sector_map.json
    → 批量调用 Tencent GT API 获取各行业代表股实时涨跌幅
    → 按行业加权计算涨跌幅
    → 排序输出前十行业
```

**依赖：** 本地区域映射已完成（Orobas 数据计划 P2）✅
**成本：** 0 元（Tencent GT API 免费）

### 2.2 组合诊断（收藏家+Orobas）

**现状：** 收藏家存了客户的持仓品种和比例，Orobas 可查各标的最新行情。
**目标：** 能基于客户持仓做简单的结构分析（行业集中度、单票占比、浮盈浮亏）。

**实现路径：**
```
用户@玄武"分析我的持仓结构"
    → 收藏家拉持仓清单
    → Orobas 查各标的最新行情 + 所属行业
    → 组装：行业分布饼图、集中度、浮盈汇总
    → 玄武回复群聊
```

**依赖：** 收藏家已有数据 + Orobas 行情 ✅
**成本：** 0 元

## P3 — 数据深度（替代方案）

### 3.1 web 抓取财务数据

**问题：** Tushare 财务数据（PE/PB/ROE/营收/利润）需 2000 分。
**替代：** 从公开页面 web_fetch/爬取。

**可行性评估：**
- 东方财富、新浪财经等公开页面包含 PE、PB、ROE 等关键指标
- 不需要高频抓取（财务数据按季度更新），爬一次本地存 CSV
- 可以作为"手动更新"的数据源，不追求实时

**考虑方向：**
- 在收藏家或 Orobas 工作区添加一个脚本 `scripts/fetch_financials.py`
- 需要时手动运行，或者每周 cron 拉一次
- 数据存本地 CSV / SQLite，委员分析时查询本地数据

**注意事项：**
- 需要测试各数据源的反爬策略
- 之前测试过 Sina Finance = JS 动态渲染，East Money push2 = IP 被拦
- 可能需要多个备选源降级

**成本：** 0 元（但需要额外开发时间）

## 网关重启纪律（2026-05-09 董事会批准）

### 核心原则
> **只有玄武主席可以执行网关重启**，其他任何 Agent 禁止自行重启。

### 规则详情
1. **重启唯一授权**：玄武是唯一有权执行 `gateway restart` 的 Agent。
2. **委员发现需重启时**：向玄武报告原因，玄武评估后统一执行。委员不得自行重启。
3. **玄武执行重启前**：玄武须确保所有正在进行的 Agent 间通信已完成，再执行重启。
4. **委员分析并行**：委员间的分析互相独立，玄武可并行召集多位委员，无需串行等待。

### 原因
2026-05-09 出现的问题：Orobas 初始化时触发网关重启，打断了玄武同时向其他委员发出的补充规则消息。

### 数据查询纪律

向 Orobas 查询数据时，**不依赖 agents_list**，直接按 MEMORY.md 中的 sessionKey 盲发：

```
sessions_send("agent:orobas:main", "查一下工业富联+方正科技最新数据")
```

### 委员回复收集（轮询模式替代 sessions_yield）

1. sessions_send 发出议题后，等待 60-90 秒
2. 用 sessions_history 逐个拉取委员最新回复
3. 已回复的委员 → 提取分析内容
4. 未回复的委员 → 再等待 30 秒后重试
5. 超时 2 分钟 → 报告董事会
